# 任务：Wiki 健康检查 (Lint)

> 你是记忆质量审计员。检查整个记忆系统的健康度并输出诊断报告。

## 检查项

### 1. 死链检测
- L1_INDEX.md 中指向的 Topic 文件是否都存在？
- Topic 文件中的交叉引用（`→ 另见 topics/xxx.md`）是否有效？

### 2. 孤儿页面
- topics/ 下是否有 L1_INDEX 未引用的文件？
- 如果有，建议补充到索引中或标记为归档候选。

### 3. 矛盾检测
- 同一事实在多个 Topic 中的描述是否一致？
- 是否有标记了 `[矛盾]` 但未解决的冲突？

### 4. 信息空洞
- 用 `[待填充]` 标记的条目是否仍然为空？
- 是否有重要概念被提及但没有自己的 Topic 页面？

### 5. 交叉引用完整性
- 相关的 Topic 之间是否有互相引用？
- 是否有明显应该关联但缺少交叉引用的主题？

## 输出格式

```markdown
## Lint 报告 - {日期}

### 🔴 错误（必须修复）
- [死链] L1_INDEX 引用了 topics/xxx.md，但文件不存在
- [矛盾] user_profile 与 preferences 中关于 X 的描述不一致

### 🟡 警告（建议修复）
- [孤儿] topics/xxx.md 未在 L1_INDEX 中引用
- [空洞] user_profile 中 "核心目标" 仍为 [待填充]

### 🟢 建议（可选优化）
- [交叉引用] topics/a.md 和 topics/b.md 有关联，建议互相引用
- [新页面] "OpenClaw" 在多处被提及，建议创建独立 Topic

### 📊 健康度评分
- 总计：X 个错误，Y 个警告，Z 个建议
- 评分：{良好/需要关注/亟需整理}
```
